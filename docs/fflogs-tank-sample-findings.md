# FFLogs 紫色以上双 T 样本观察

## 采样口径

- 来源：FFLogs v2 GraphQL，本地采集，不提交原始样本。
- 副本：阿卡狄亚登天斗技场四层，先采 encounter 104 / 105。
- 排行：按坦克 HPS 排行榜前 100 候选。
- 过滤：必须击杀；双 T 在同一战斗内 HPS `rankPercent >= 75`；匿名/无报告/未击杀/百分位缺失样本丢弃。
- 当前样本：91 条可用样本，其中 104 为 73 条，105 为 18 条。

## 样本分布

- 职业主视角：PLD 19、WAR 12、DRK 31、GNB 29。
- 搭档组合：DRK+PLD 33、PLD+GNB 23、PLD+WAR 14、DRK+GNB 13、WAR+GNB 4、DRK+WAR 3、DRK+DRK 1。
- 高频技能：雪仇、铁壁、圣盾阵、至黑之夜、刚玉之心、献奉、干预、圣光幕帘、暗黑布道、极光。

## 观察结论

1. 紫色以上双 T 样本不会把长 CD 大减按时间顺序交给前几个大伤害。长 CD 通常保留给后续明确高危窗口。
2. 短 CD 和目标支援是覆盖率核心。圣盾阵、至黑之夜、刚玉之心、血气、勇猛、干预、献奉、极光会高频填充单体压力。
3. AOE 主要使用雪仇和职业特色团减，例如暗黑布道、光之心、圣光幕帘、武装戍卫、摆脱。
4. 死刑常见结构是短 CD/支援减伤加一层硬减。只有致命窗口、双 T 分摊/分散或无敌逃课时，才更积极叠多层资源。
5. 跨给搭档很常见，尤其是 DRK+PLD、PLD+GNB、DRK+GNB、PLD+WAR。算法不能只让受击 T 开自己的技能。
6. DRK 的至黑之夜要积极使用，但需要限制同一分钟内的次数，避免破坏 MP 循环。当前规则为每 60 秒最多 4 次。

## 已落地到算法

- 高危事件优先处理，再回填普通事件，避免早期普通伤害抢走后续死刑资源。
- 长 CD 个人减伤如果后续 CD 内存在更高危窗口，会被降权保留。
- AOE 优先雪仇和特色团减；普通 AOE 降低个人硬减和无敌权重。
- 平 A 合并成压力窗口，优先短 CD/目标支援，不让团减或无敌抢资源。
- 单体/双 T 死刑优先短 CD、支援减伤和一层硬减组合；非致命窗口降低同类硬减过度叠加。
- 双 T 目标会同时考虑 MT/ST 双方技能，包括可跨给搭档的技能。
- 根据紫色样本给常见技能与搭档组合加权。

## 当前限制

- 采样仍是启发式分析，不是完整数学最优解。
- 目前还没有按具体 Boss 技能名建立固定机制库，仍依赖导入时间轴事件的类型、目标、伤害属性与严重程度。
- 样本中 encounter 105 较少，后续需要继续补采。
- HPS 高不完全等价于团队最稳打法，算法会偏向覆盖率与资源利用，但仍保留玩家手动调整空间。
