# FFLogs 双 T HPS95+ 样本观察

## 采样口径

- 来源：FFLogs v2 GraphQL，本地采集，不提交原始样本。
- 副本：阿卡狄亚登天斗技场四层，先以 encounter 104 小样本跑通。
- 排行：按坦克 HPS 排行榜前 100 候选。
- 过滤：必须击杀；双 T 在同一战斗内 HPS `rankPercent >= 95`；匿名/无报告/未击杀/百分位缺失样本丢弃。
- 限制：本轮被 FFLogs API 429 限流，完整“每职业 100 条”需要分批补采。当前先用已通过过滤的 6 条样本落地第一版策略。

## 初步结论

1. 高 HPS 样本不会把大减伤按时间顺序交给前几个高伤害。长 CD 会保留给明确高危窗口，普通压力更多交给短 CD 和目标支援。
2. 短 CD/目标支援使用频率很高，例如圣盾阵、至黑之夜、献奉、刚玉之心、极光。它们是提升覆盖率和 HPS 的主要来源。
3. 双 T 配合里经常出现跨给搭档的减伤，目标不是“谁被打谁只开自己的”，而是让每个受击窗口至少有一层短 CD 或支援减伤。
4. AOE 主要依赖雪仇和特色团减，个人硬减不应优先分配给普通 AOE。
5. 死刑窗口常见组合是“短 CD/目标支援 + 一层硬减/专属减伤”，除非致命窗口或双 T 特殊处理，否则不鼓励多层硬减叠满。
6. DRK 的至黑之夜要积极用，但按 60 秒窗口限制次数。本项目先限制为每 60 秒最多 4 次，避免影响 MP 循环。

## 已落地到算法

- 先处理高危事件，再回填普通事件，避免前面普通伤害抢走后面死刑资源。
- CD 检查改成绝对时间冲突，而不是只看上一条使用记录。
- 非 T 随机机制不进入坦克减伤规划。
- AOE 优先团减/雪仇，平 A 优先短 CD。
- 单体/双 T 死刑优先短 CD、支援减伤和一层硬减组合。
- 长 CD 无敌只在高危死刑或用户开启“优先无敌逃课”时才积极使用。
- DRK 至黑之夜加入每 60 秒最多 4 次限制。
